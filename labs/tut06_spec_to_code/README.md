# Tutorial 6 — Spec to Code

The Field Repair Tracker `assign_job` specification used to drive AI code
generation in Tutorial 6 Activity 3.

## Files

| File | Purpose |
|---|---|
| `spec_assign_job.md` | The specification document fed to the AI agent |
| `sample_acceptance.feature` | Sample Gherkin acceptance criteria from Activity 1 |

## How to use

Open the spec in an AI Assistant session and prompt it as described in
Activity 3 Step 2 to generate `src/domain/repair_job.py`,
`src/repository/job_repository.py`, `src/service/job_service.py`,
and `src/api/job_router.py`.

The tutorial does not ship a reference implementation — the activity is
specifically about reviewing what the AI produces against the
specification.
