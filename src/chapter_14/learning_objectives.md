## Learning Objectives

By the end of this chapter, you will be able to:

1. Describe the ML data lifecycle from collection through to the training artifact, identifying the engineering failure modes at each stage.
2. Explain what it means for a dataset to be a versioned engineering artifact, and why this differs from simply storing files on disk.
3. Apply the five dimensions of dataset quality — validity, accuracy, completeness, consistency, and timeliness — to evaluate a dataset for use in a production ML application.
4. Distinguish between image-level and patient-level data splitting, and explain the mechanism by which image-level splitting introduces leakage that inflates apparent performance.
5. Describe what an ML run management system records and why each element is necessary for a production ML application to be reproducible and auditable.
6. Map the components of MLflow to the engineering problems they solve in building a production ML application.
