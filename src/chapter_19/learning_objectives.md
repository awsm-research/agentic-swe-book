## Learning Objectives

By the end of this chapter, you will be able to:

1. Distinguish between a prompt and a context window, and explain what *context engineering* requires of a system designer.
2. Select the appropriate prompting strategy — zero-shot, few-shot, chain-of-thought, self-consistency, or ReAct — for a given task type and justify the selection.
3. Explain how reasoning models differ from standard models and identify the conditions under which each is the better choice.
4. Apply context engineering principles to system prompt design, retrieval placement, conversation history management, and tool schema authoring.
5. Treat prompts as versioned engineering artefacts: describe the prompt lifecycle, define regression test criteria, and identify prompt injection attack vectors.
6. Apply the MedChat case study to evaluate how prompt and context engineering decisions affect clinical safety and system reliability.
