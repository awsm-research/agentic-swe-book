## Learning Objectives

By the end of this chapter, you will be able to:

1. Define an autonomous agent and distinguish it clearly from a chatbot and a copilot, describing how the locus of control differs across each.
2. Explain the ReAct loop — Reason, Act, Observe, repeat — and describe why this pattern is the architectural foundation of practical agent design.
3. Classify agent tools by their read/write distinction and explain the safety and idempotency requirements that apply to write tools.
4. Describe the four categories in the agent memory taxonomy and explain when each is appropriate.
5. Explain when explicit planning improves agent performance and when it adds unnecessary latency.
6. Describe LangGraph as a state machine for agent design, including the roles of nodes, edges, and conditional routing.
7. Identify the four primary agent failure modes and describe the engineering controls that address each.
8. Map the ReAct loop, tool taxonomy, and memory design onto the MedAgent clinical case study.
